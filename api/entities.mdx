# Entity Schemas

The Firedancer Frontend uses Zod schemas to define and validate all data entities received from the validator. These schemas ensure type safety and runtime validation of WebSocket messages.

## Core Entities

### Client

Identifies the validator client type.

<ResponseField name="clientSchema" type="enum">
  Validator client type: `Frankendancer` or `Firedancer`
</ResponseField>

```typescript
export const clientSchema = z.enum(["Frankendancer", "Firedancer"]);
export const ClientEnum = clientSchema.enum;
```

### Cluster

Identifies the Solana cluster the validator is connected to.

<ResponseField name="clusterSchema" type="enum">
  Supported clusters:
  - `development`
  - `mainnet-beta`
  - `devnet`
  - `testnet`
  - `pythtest`
  - `pythnet`
  - `unknown`
</ResponseField>

```typescript
export const clusterSchema = z.enum([
  "development",
  "mainnet-beta",
  "devnet",
  "testnet",
  "pythtest",
  "pythnet",
  "unknown",
]);
```

### Topic Schema

Defines the available WebSocket message topics using discriminated unions.

<ResponseField name="topicSchema" type="discriminatedUnion">
  Available topics:
  - `summary` - General validator information
  - `epoch` - Epoch-related data
  - `gossip` - Gossip network statistics
  - `peers` - Peer validator information
  - `slot` - Slot performance data
  - `block_engine` - Block engine connection status
</ResponseField>

```typescript
export const topicSchema = z.discriminatedUnion("topic", [
  summaryTopicSchema,
  epochTopicSchema,
  gossipTopicSchema,
  peersTopicSchema,
  slotTopicSchema,
  blockEngineTopicSchema,
]);
```

## Tile Entities

### Tile Type

Defines all available tile types in the Firedancer validator architecture.

<ResponseField name="tileTypeSchema" type="enum">
  Core tiles: `sock`, `net`, `quic`, `bundle`, `verify`, `dedup`, `resolv`/`resolh`, `pack`, `execle`/`bank`, `poh`/`pohh`, `shred`, `store`
  
  Snapshot tiles: `snapct`, `snapld`, `snapdc`, `snapin`
  
  Shred tiles: `netlnk`, `metric`, `ipecho`, `gossvf`, `gossip`, `repair`, `replay`, `execrp`, `tower`, `txsend`, `sign`, `rpc`, `gui`
  
  Other tiles: `http`, `plugin`, `cswtch`, `genesi`, `diag`, `event`
</ResponseField>

```typescript
export const tileTypeSchema = z.enum([
  "sock", "net", "quic", "bundle", "verify", "dedup",
  "resolv", // Firedancer
  "resolh", // Frankendancer
  "pack",
  "execle", // Firedancer
  "bank",   // Frankendancer
  "poh",    // Firedancer
  "pohh",   // Frankendancer
  "shred", "store",
  // ... additional tiles
]);
```

## Slot Entities

### Slot Level

Defines the confirmation level of a slot.

<ResponseField name="slotLevelSchema" type="enum">
  - `incomplete` - Slot is still being processed
  - `completed` - Slot processing completed
  - `optimistically_confirmed` - Slot is optimistically confirmed
  - `rooted` - Slot has been rooted
  - `finalized` - Slot is finalized
</ResponseField>

```typescript
export const slotLevelSchema = z.enum([
  "incomplete",
  "completed",
  "optimistically_confirmed",
  "rooted",
  "finalized",
]);
```

### Slot Publish

Core slot performance data published when a slot completes.

<ResponseField name="slotPublishSchema" type="object">
  <ResponseField name="slot" type="number" required>
    Slot number
  </ResponseField>
  <ResponseField name="mine" type="boolean" required>
    Whether this validator was the leader
  </ResponseField>
  <ResponseField name="skipped" type="boolean" required>
    Whether the slot was skipped
  </ResponseField>
  <ResponseField name="level" type="SlotLevel" required>
    Confirmation level
  </ResponseField>
  <ResponseField name="success_nonvote_transaction_cnt" type="number | null">
    Count of successful non-vote transactions
  </ResponseField>
  <ResponseField name="failed_nonvote_transaction_cnt" type="number | null">
    Count of failed non-vote transactions
  </ResponseField>
  <ResponseField name="success_vote_transaction_cnt" type="number | null">
    Count of successful vote transactions
  </ResponseField>
  <ResponseField name="failed_vote_transaction_cnt" type="number | null">
    Count of failed vote transactions
  </ResponseField>
  <ResponseField name="priority_fee" type="bigint | null">
    Total priority fees in lamports
  </ResponseField>
  <ResponseField name="transaction_fee" type="bigint | null">
    Total transaction fees in lamports
  </ResponseField>
  <ResponseField name="tips" type="bigint | null">
    Total tips in lamports
  </ResponseField>
  <ResponseField name="max_compute_units" type="number | null">
    Maximum compute units available
  </ResponseField>
  <ResponseField name="compute_units" type="number | null">
    Compute units consumed
  </ResponseField>
  <ResponseField name="duration_nanos" type="number | null">
    Slot duration in nanoseconds
  </ResponseField>
  <ResponseField name="completed_time_nanos" type="bigint | null">
    Completion timestamp in nanoseconds
  </ResponseField>
  <ResponseField name="vote_latency" type="number | null">
    Vote latency in slots
  </ResponseField>
</ResponseField>

```typescript
export const slotPublishSchema = z.object({
  slot: z.number(),
  mine: z.boolean(),
  skipped: z.boolean(),
  level: slotLevelSchema,
  success_nonvote_transaction_cnt: z.number().nullable(),
  failed_nonvote_transaction_cnt: z.number().nullable(),
  success_vote_transaction_cnt: z.number().nullable(),
  failed_vote_transaction_cnt: z.number().nullable(),
  priority_fee: z.coerce.bigint().nullable(),
  transaction_fee: z.coerce.bigint().nullable(),
  tips: z.coerce.bigint().nullable(),
  max_compute_units: z.number().nullable(),
  compute_units: z.number().nullable(),
  duration_nanos: z.number().nullable(),
  completed_time_nanos: z.coerce.bigint().nullable(),
  vote_latency: z.number().nullable(),
});
```

## Transaction Waterfall

Tracks transaction flow through the validator pipeline.

### Transaction Waterfall In

<ResponseField name="txnWaterfallInSchema" type="object">
  <ResponseField name="pack_cranked" type="number">
    Transactions processed from pack queue
  </ResponseField>
  <ResponseField name="pack_retained" type="number">
    Transactions retained in pack queue
  </ResponseField>
  <ResponseField name="resolv_retained" type="number">
    Transactions retained in resolver
  </ResponseField>
  <ResponseField name="quic" type="number">
    Transactions from QUIC
  </ResponseField>
  <ResponseField name="udp" type="number">
    Transactions from UDP
  </ResponseField>
  <ResponseField name="gossip" type="number">
    Transactions from gossip
  </ResponseField>
  <ResponseField name="block_engine" type="number">
    Transactions from block engine
  </ResponseField>
</ResponseField>

### Transaction Waterfall Out

<ResponseField name="txnWaterfallOutSchema" type="object">
  Tracks transaction drop/failure reasons at each stage:
  
  <ResponseField name="net_overrun" type="number">Network overrun drops</ResponseField>
  <ResponseField name="quic_overrun" type="number">QUIC overrun drops</ResponseField>
  <ResponseField name="verify_failed" type="number">Signature verification failures</ResponseField>
  <ResponseField name="dedup_duplicate" type="number">Duplicate transactions</ResponseField>
  <ResponseField name="pack_expired" type="number">Expired in pack queue</ResponseField>
  <ResponseField name="block_success" type="number">Successfully executed</ResponseField>
  <ResponseField name="block_fail" type="number">Execution failures</ResponseField>
  
  And 20+ additional failure categories...
</ResponseField>

```typescript
export const txnWaterfallSchema = z.object({
  in: txnWaterfallInSchema,
  out: txnWaterfallOutSchema,
});
```

## Summary Schema

The summary schema is a discriminated union that combines topic and key-value pairs for all validator metrics.

```typescript
export const summarySchema = z.discriminatedUnion("key", [
  summaryTopicSchema.extend({ key: z.literal("ping"), value: z.null(), id: z.number() }),
  summaryTopicSchema.extend({ key: z.literal("version"), value: versionSchema }),
  summaryTopicSchema.extend({ key: z.literal("cluster"), value: clusterSchema }),
  // ... 40+ additional key-value pairs
]);
```
