The Startup Progress feature provides real-time monitoring of your validator's initialization sequence, from gossip joining through snapshot loading and network catchup. This overlay appears automatically during validator startup and disappears once the validator is fully operational.

## Overview

The Startup Progress component (`src/features/StartupProgress/index.tsx`) acts as a wrapper that displays progress information and optionally blurs the main dashboard:

```tsx
export default function StartupProgress({ children }: PropsWithChildren) {
  const showStartupProgress = useAtomValue(showStartupProgressAtom);
  const client = useAtomValue(clientAtom);
  if (!client) return null;

  return client === ClientEnum.Firedancer ? (
    <>
      <FiredancerBody />
      <div>{children}</div>
    </>
  ) : (
    <>
      <Body />
      <div
        className={clsx(styles.container, {
          [styles.blur]: showStartupProgress,
        })}
      >
        {children}
      </div>
    </>
  );
}
```

## Client-Specific Implementations

### Firedancer Startup

Firedancer validators display a comprehensive startup screen with detailed phase tracking:

```tsx src/features/StartupProgress/Firedancer/Body.tsx
export default function Body() {
  const setShowStartupProgress = useSetAtom(showStartupProgressAtom);
  const phase = useAtomValue(bootProgressPhaseAtom);

  // Automatically close when running
  useEffect(() => {
    setShowStartupProgress(phase !== "running");
  }, [setShowStartupProgress, phase]);

  return (
    <>
      {phase && <BootProgressContent phase={phase} />}
      <Logo />
    </>
  );
}
```

### Frankendancer Startup

Frankendancer uses a simplified startup display without the detailed phase tracking available in Firedancer.

## Boot Phases

The startup sequence progresses through several distinct phases:

```tsx
export enum BootPhaseEnum {
  joining_gossip = "joining_gossip",
  loading_full_snapshot = "loading_full_snapshot",
  loading_incremental_snapshot = "loading_incremental_snapshot",
  catching_up = "catching_up",
  running = "running",
}
```

#### Replay Performance

The catchup screen shows real-time performance metrics:

```tsx
function CatchupMetrics() {
  const replaySlot = useAtomValue(replaySlotAtom);
  const targetSlot = useAtomValue(networkSlotAtom);
  const slotsBehind = targetSlot - replaySlot;
  
  const replayRate = useReplayRate(); // slots per second
  const eta = slotsBehind / replayRate;
  
  return (
    <Flex direction="column" gap="2">
      <Stat label="Slots Behind" value={slotsBehind.toLocaleString()} />
      <Stat label="Replay Rate" value={`${replayRate.toFixed(1)} slots/s`} />
      <Stat label="ETA" value={formatDuration(eta)} />
    </Flex>
  );
}
```

## Startup Progress Container

The container component manages the startup overlay:

```tsx
function BootProgressContent({ phase }: BootProgressContentProps) {
  const setBootProgressContainerEl = useSetAtom(bootProgressContainerElAtom);
  const showStartupProgress = useAtomValue(showStartupProgressAtom);
  const isStartupProgressExpanded = useAtomValue(isStartupProgressExpandedAtom);

  const phaseClass = phase ? classNames[phase] : "";
  const isNarrow = useMedia("(max-width: 750px)");

  return (
    <Box
      ref={(el: HTMLDivElement) => setBootProgressContainerEl(el)}
      overflowY="auto"
      className={clsx(styles.container, phaseClass, {
        [styles.collapsed]: !showStartupProgress || !isStartupProgressExpanded,
      })}
    >
      <Header isStartup />
      
      <Flex
        direction="column"
        width="100%"
        maxWidth={appMaxWidth}
        mx="auto"
        px={isNarrow ? "20px" : "89px"}
      >
        {(phase === BootPhaseEnum.loading_full_snapshot ||
          phase === BootPhaseEnum.loading_incremental_snapshot) && <Snapshot />}
        {phase === BootPhaseEnum.catching_up && <CatchingUp />}
      </Flex>
    </Box>
  );
}
```

## Automatic State Management

The startup progress state is managed automatically:

```tsx
const showStartupProgressAtom = atom((get) => {
  const phase = get(bootProgressPhaseAtom);
  return phase !== undefined && phase !== "running";
});
```

No manual intervention required - the overlay appears and disappears based on validator state.

## Progress Persistence

If you refresh the page during startup:

- Progress state is retrieved from validator
- Display resumes at current phase
- No data is lost
- Estimates recalculate based on current position

## Error Handling

If startup encounters errors:

### Download Failures

- Automatic retry with exponential backoff
- Fallback to alternative snapshot sources
- Error message with retry countdown

### Verification Failures

- Snapshot hash mismatch detection
- Automatic re-download from different source
- Warning notifications

### Network Issues

- Connection timeout detection
- Peer reconnection attempts
- Network diagnostic information

## Comparison to Previous Starts

The UI may show comparisons to previous startup times:

```tsx
function StartupComparison() {
  const currentDuration = useCurrentStartupDuration();
  const avgPreviousDuration = useAverageStartupDuration();
  
  const comparison = currentDuration - avgPreviousDuration;
  const isSlower = comparison > 0;
  
  return (
    <Text color={isSlower ? "red" : "green"}>
      {isSlower ? "Slower" : "Faster"} than average by {formatDuration(Math.abs(comparison))}
    </Text>
  );
}
```

## Troubleshooting

### Stuck at Joining Gossip

1. Verify network connectivity
2. Check firewall rules for gossip port
3. Ensure entrypoints are reachable
4. Verify system clock is synchronized

### Slow Snapshot Download

1. Check network bandwidth
2. Try different snapshot source
3. Verify disk write speed
4. Consider using local snapshot cache

### Catchup Taking Too Long

1. Check CPU utilization
2. Verify sufficient memory available
3. Monitor disk I/O performance
4. Consider hardware upgrades if consistently slow

### Startup Failures

If startup repeatedly fails:
1. Check validator logs for specific errors
2. Verify accounts DB integrity
3. Ensure sufficient disk space
4. Try deleting ledger and restarting fresh
