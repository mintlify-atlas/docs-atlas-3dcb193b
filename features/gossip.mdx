<Note>

</Note>

The Gossip page provides comprehensive monitoring of your validator's participation in Solana's gossip protocol, including peer tracking, message statistics, traffic analysis, and network health metrics.

## Access Control

The gossip route includes automatic client detection and redirect:

```tsx src/routes/gossip.tsx
export const Route = createFileRoute("/gossip")({
  component: Gossip,
  beforeLoad: ({ context, location }) => {
    if (store.get(clientAtom) === ClientEnum.Frankendancer) {
      throw redirect({
        to: "/",
      });
    }
  },
});
```

## Page Structure

The Gossip feature (`src/features/Gossip/index.tsx`) organizes network data into several key sections:

```tsx
export default function Gossip() {
  const client = useAtomValue(clientAtom);
  const networkStats = useAtomValue(gossipNetworkStatsAtom);
  const [dbNetworkStats] = useDebounce(networkStats, 5_000, {
    leading: true,
    maxWait: 5_000,
  });

  const health = networkStats?.health;
  const storage = networkStats?.storage;

  if (client === ClientEnum.Frankendancer) return;
  if (!health || !storage || !dbNetworkStats) return;

  return (
    <Flex gap="30px" direction="column" align="stretch" justify="center" height="100%">
      {/* Traffic TreeMaps */}
      <Flex gapX="30px" gapY={rowGap} wrap="wrap">
        <TrafficTreeMap networkTraffic={dbNetworkStats.ingress} label="Ingress" />
        <TrafficTreeMap networkTraffic={dbNetworkStats.egress} label="Egress" />
      </Flex>

      {/* Storage and Message Stats */}
      <Flex gap="30px" wrap="wrap">
        <Flex direction="column" flexGrow="1" flexBasis="0" gap={rowGap}>
          <StorageStatsCharts storage={storage} />
          <StorageStatsTable storage={storage} />
        </Flex>
        <Flex direction="column" flexGrow="1" flexBasis="0" gap={rowGap}>
          <StakeStatsChart />
          <MessageStatsTable messages={networkStats.messages} />
        </Flex>
      </Flex>

      {/* Health and Peer Data */}
      <GossipHealth health={health} />
      <PeerTable />
    </Flex>
  );
}
```

## Rate Display

Bandwidth rates are displayed with automatic unit conversion:

```tsx src/features/Gossip/RateTableCell.tsx
function formatBytesAsBits(bytes: number) {
  const bits = bytes * 8;
  
  if (bits >= 1_000_000_000) {
    return {
      value: (bits / 1_000_000_000).toFixed(2),
      unit: "Gbps"
    };
  } else if (bits >= 1_000_000) {
    return {
      value: (bits / 1_000_000).toFixed(2),
      unit: "Mbps"
    };
  } else if (bits >= 1_000) {
    return {
      value: (bits / 1_000).toFixed(2),
      unit: "Kbps"
    };
  }
  
  return {
    value: bits.toFixed(0),
    unit: "bps"
  };
}
```

## Performance Optimizations

### Debounced Updates

Network statistics use debouncing to prevent excessive re-renders:

```tsx
const [dbNetworkStats] = useDebounce(networkStats, 5_000, {
  leading: true,
  maxWait: 5_000,
});
```

### Lazy Loading

The peer table loads data on-demand as you scroll:

```tsx
const onRangeChanged = useCallback(
  ({ startIndex, endIndex }: ListRange) => {
    const start = Math.max(0, startIndex - buffer);
    const end = Math.min(
      endIndex + buffer,
      size > 0 ? size - 1 : endIndex + buffer,
    );
    query(start, end);
  },
  [query, size],
);
```

### Memoization

Expensive calculations are memoized to prevent recalculation:

```tsx
const components = useMemo(
  () => getComponents(colIds, colWidths),
  [colIds, colWidths],
);
```

## Troubleshooting

### No Peer Data

If the peer table is empty:
1. Verify gossip is enabled in your validator config
2. Check firewall rules allow gossip port (default: 8001)
3. Ensure sufficient time for peer discovery (30-60 seconds)

### High Message Rates

If seeing excessive message rates:
1. Check for duplicate shred reports (potential fork)
2. Verify network connectivity quality
3. Monitor CPU utilization in [Tile Metrics](/features/overview#live-tile-metrics)

### Storage Warnings

If storage capacity is near limits:
1. Increase gossip storage configuration
2. Check for memory leaks in validator
3. Monitor record expiration rates
